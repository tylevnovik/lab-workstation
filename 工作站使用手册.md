## 课题组工作站使用手册

本文档是课题组公共工作站的使用规范，涵盖文件夹结构、权限管理、Python 环境配置及日常使用流程。所有使用本工作站的同学请仔细阅读并遵守。

---

### 一、工作站概况

- **操作系统**：Windows Server 2022
- **用途**：课题组公共数据处理、项目协作、报告撰写
- **访问范围**：仅限课题组内部成员（通过 Windows 账户权限控制）

---

### 二、文件夹结构规范

工作站的数据存储分为**三个区域**：

- **`D:\GroupData\_公共\`**（组间公共区）— 所有导师组的所有成员都能看见和读写。放跨组共享的数据、公共工具和模板。
- **`D:\GroupData\导师名\`**（组内公共区）— 只有该导师组内的成员可见。放组内协作的数据和报告。
- **`D:\Users\用户名\`**（个人目录）— 每个账户独享，其他人看不到也进不去。私有数据、草稿放这里。

#### 2.1 文件夹结构

```
D:\GroupData\
│
├── _公共\                                  ← 组间公共：所有组所有成员可见
│   ├── _使用手册\                          ← 本文档及管理工具
│   │   ├── 工作站使用手册.md
│   │   └── Manage-LabAccounts.ps1
│   ├── 工具与模板\                         ← 跨组共享的脚本、模板
│   │   ├── 报告模板\
│   │   ├── 数据清洗脚本\
│   │   └── 可视化工具\
│   └── 跨组共享数据\                       ← 需要多组协作的数据
│
├── 张老师\                                 ← 张老师组的组内公共区（仅本组可见）
│   ├── 01_人才类数据\
│   ├── 02_温故知新数据\
│   ├── 03_科技报告\
│   ├── 04_资政报告\
│   ├── 05_项目资料\
│   └── 99_归档\
│
├── 李老师\                                 ← 李老师组的组内公共区（仅本组可见）
│   ├── 01_人才类数据\
│   ├── 02_温故知新数据\
│   ├── 03_科技报告\
│   ├── 04_资政报告\
│   ├── 05_项目资料\
│   └── 99_归档\
│
└── ...更多导师组...
```

#### 2.2 个人目录

每个账户创建时，管理工具会自动在 `D:\Users\` 下建立同名个人目录并做权限隔离。

```
D:\Users\
├── zhangsan\                   ← 张三（张老师组），仅本人和管理员可访问
├── lisi\                       ← 李四（李老师组），仅本人和管理员可访问
└── ...
```

#### 2.3 命名规则

- **导师组文件夹**：直接用导师姓名命名，如 `张老师`、`李老师`。
- **组内类别文件夹**：以 `两位数编号_类别名称` 命名，如 `01_人才类数据`，各组统一。
- **项目/数据子文件夹**：用简洁的名称命名，如 `2025国家自然科学基金`，避免特殊字符。
- **归档文件夹**：格式为 `年份_项目名`，如 `2024_人才政策研究`。

#### 2.4 文件夹创建流程

1. **新增导师组**：需管理员在管理工具中操作，会自动创建组文件夹、安全组和标准类别子文件夹。
2. **组内新建项目/数据子文件夹**：组内成员可自行在对应类别下创建。
3. **`_公共` 区写入**：放入组间公共数据前请确认数据确实是跨组共享的，避免误放组内数据。

### 三、权限管理方案

#### 3.1 安全组设计

权限通过**两级安全组**实现隔离：

```
Lab_All（全员组）             ← 所有工作站用户都在此组
├── 访问 _公共\ 文件夹
│
├── Lab_张老师（张老师组）    ← 张老师及其学生
│   └── 访问 GroupData\张老师\ 文件夹
│
├── Lab_李老师（李老师组）    ← 李老师及其学生
│   └── 访问 GroupData\李老师\ 文件夹
│
└── ...
```

每位用户在创建账户时指定导师，系统会自动将其加入 `Lab_All` 和对应的 `Lab_导师名` 两个安全组。

**NTFS 权限分配表：**

| 文件夹 | Lab_All | Lab_张老师 | Lab_李老师 | Administrators |
|--------|---------|-----------|-----------|----------------|
| `D:\GroupData\` | 遍历文件夹 | 遍历文件夹 | 遍历文件夹 | 完全控制 |
| `D:\GroupData\_公共\` | 修改（递归） | — | — | 完全控制 |
| `D:\GroupData\张老师\` | 无 | 修改（递归） | 无 | 完全控制 |
| `D:\GroupData\李老师\` | 无 | 无 | 修改（递归） | 完全控制 |
| `D:\Users\zhangsan\` | 无 | 无 | 无 | 完全控制 + zhangsan 本人 |

"遍历文件夹"权限的意思是：用户可以看到 GroupData 根目录下有哪些文件夹，但无法进入没有授权的文件夹。

#### 3.2 创建安全组

管理工具会自动处理安全组的创建，以下是其内部逻辑（手动操作时参考）：

```powershell
# 创建全员组
net localgroup Lab_All /add

# 创建导师组
net localgroup Lab_张老师 /add

# 将用户加入两个组
net localgroup Lab_All zhangsan /add
net localgroup Lab_张老师 zhangsan /add
```

#### 3.3 SMB 共享配置

`D:\GroupData` 共享给 `Lab_All` 组，赋予"更改"权限。具体的文件夹级隔离由 NTFS 权限控制。

#### 3.4 禁用匿名访问

确保服务器"高级共享设置"中：开启密码保护共享、关闭 Guest 账户、启用 `网络访问: 不允许匿名枚举 SAM 账户和共享`。

#### 3.5 关于加密

- **BitLocker**（推荐）：对整个数据盘加密，防止物理拆盘泄露，日常使用无感。
- **EFS**：对特定文件夹加密，仅授权证书可解密。适合特别敏感的数据，但管理复杂度更高。

```powershell
# 启用 BitLocker
Enable-BitLocker -MountPoint "D:" -EncryptionMethod Aes256 -RecoveryPasswordProtector
```

#### 3.6 网络访问方式

```powershell
# 映射网络驱动器
net use Z: \\服务器IP\GroupData /persistent:yes /user:你的账户
```

### 四、Python 环境管理规范

为了避免各项目的依赖互相冲突，工作站上的 Python 环境采用**统一管理 + 项目隔离**的策略。

#### 4.1 基础工具安装

工作站上只需安装一个核心工具：

**uv — 一站式 Python 管理工具**

uv 是新一代 Python 工具链，可以一站式完成 Python 版本安装、虚拟环境创建和包管理，速度极快，无需再安装 Conda 或 pip。

```powershell
# 安装 uv
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"

# 验证安装
uv --version

# 验证 Python 管理能力（会自动下载管理 Python）
uv python list
```

**PyCharm**

安装一套 PyCharm Professional（学校可能有教育授权）或 Community 版。配置好 uv 后，每个项目打开时选择对应的解释器即可。

#### 4.2 环境管理流程

整体思路：**uv 管一切 — Python 版本、虚拟环境、包安装全部由 uv 完成**。

```
uv python install 3.11     ← 用 uv 安装需要的 Python 版本
    │
    ├── uv venv .venv      ← 每个项目用 uv 创建独立虚拟环境
    │   └── .venv\         ← 环境文件夹放在项目目录下
    │
    └── uv venv .venv      ← 另一个项目的独立环境
        └── .venv\
```

**安装 Python 版本（首次使用或需要新版本时）：**

```powershell
# 查看可安装的 Python 版本
uv python list

# 安装指定版本
uv python install 3.11
uv python install 3.12
```

**创建新项目环境的标准流程：**

```powershell
# 1. 进入项目文件夹（在自己导师组的组内区域）
cd D:\GroupData\张老师\03_科技报告\2025国自然项目

# 2. 用 uv 创建虚拟环境（指定 Python 版本，uv 会自动下载对应版本）
uv venv .venv --python 3.11

# 3. 激活虚拟环境
.venv\Scripts\activate

# 4. 安装项目依赖
uv pip install pandas numpy matplotlib scikit-learn

# 5.（可选）导出依赖清单
uv pip freeze > requirements.txt
```

**已有项目的环境恢复：**

```powershell
# 进入项目目录，创建环境并安装依赖
cd D:\GroupData\张老师\03_科技报告\2025国自然项目
uv venv .venv --python 3.11
.venv\Scripts\activate
uv pip install -r requirements.txt
```

#### 4.3 项目文件夹规范

每个项目文件夹建议包含以下内容：

```
项目名\
├── .venv\                ← Python 虚拟环境（不要提交到版本控制）
├── .gitignore            ← 排除 .venv、__pycache__ 等
├── requirements.txt      ← 依赖清单
├── data\                 ← 项目数据（如果数据量大，可指向公共数据路径）
├── src\                  ← 源代码
├── output\               ← 输出结果（图表、报告等）
└── README.md             ← 项目说明
```

#### 4.4 PyCharm 配置

1. 打开 PyCharm → File → Settings → Project → Python Interpreter
2. 点击 Add Interpreter → Existing
3. 选择项目目录下 `.venv\Scripts\python.exe`
4. 确认后即可在 PyCharm 中正常使用该项目的虚拟环境

#### 4.5 注意事项

- **不要把虚拟环境文件夹（`.venv`）复制到别的地方**，环境是不可移植的，应该在新位置重新创建。
- **每个项目一个 `requirements.txt`**，方便其他人复现环境。
- **不要在全局安装大量包**，所有依赖都应该在虚拟环境内安装，保持系统 Python 干净。
- 如果项目需要 GPU 支持（如深度学习），在创建环境时指定 CUDA 版本的 PyTorch，例如：`uv pip install torch --index-url https://download.pytorch.org/whl/cu121`

### 五、日常使用守则

#### 5.1 数据存放原则

三个区域各有用途，按以下规则判断数据该放哪里：

| 场景 | 存放位置 |
|------|---------|
| 跨组共享的数据集、公共工具和模板 | `D:\GroupData\_公共\` |
| 本组内的项目数据、报告（其他组不需要看） | `D:\GroupData\你的导师名\对应类别\` |
| 自己正在处理的草稿、临时文件 | `D:\Users\你的用户名\` |
| 不想被别人看到的研究笔记、私人数据 | `D:\Users\你的用户名\` |
| 已结题/已毕业同学的数据 | `D:\GroupData\导师名\99_归档\` |

#### 5.2 数据命名建议

- 数据集文件：`数据名称_来源_日期.格式`，如 `人才流动数据_知网_202503.xlsx`
- 报告文件：`报告名称_版本_日期.格式`，如 `科技人才政策报告_v2_20250415.docx`
- 避免使用 `最终版`、`最终版2`、`打死不改版` 之类的命名

#### 5.3 归档流程

项目结题或同学毕业时：

1. 确认所有重要数据已整理完毕
2. 在 `D:\GroupData\导师名\99_归档\` 下创建 `年份_项目名` 文件夹
3. 将相关数据、代码、报告整体移入
4. 在归档文件夹中放一个简单的 `README.md` 说明内容

#### 5.4 禁止事项

- **禁止**在共享文件夹根目录下随意创建文件夹
- **禁止**删除他人文件夹（如有需要请联系管理员）
- **禁止**在共享文件夹中存放与课题组无关的个人文件
- **禁止**将工作站的账户密码分享给组外人员
- **建议**定期清理自己的临时文件

#### 5.5 离校流程

离开课题组时，管理员会执行"离校流程"，包括以下操作：

1. 你的个人数据将被归档到 `D:\GroupData\_公共\99_归档\离校用户_用户名_日期\` 文件夹中
2. 账户将被禁用，并从所有安全组中移除
3. 你将失去对工作站的访问权限

> **关于组内数据**：组内文件夹（`D:\GroupData\导师名\`）属于全组共享数据，离校时**不会**自动复制或移动。系统会在归档目录中生成一份交接清单（README），记录该用户曾负责的组内项目路径和状态，方便接替的同学查阅。

**请务必在离校前自行备份重要的个人数据**，归档完成后管理员可能不会保留原始个人目录。

### 六、管理员操作备忘

管理员日常操作（创建账户、分配权限、禁用账户等）统一通过可视化管理工具完成，无需记忆命令行。

#### 6.1 账户管理工具

工具文件：`D:\GroupData\_公共\_使用手册\Manage-LabAccounts.ps1`

**启动方式**：右键该文件 → "使用 PowerShell 运行"，或在管理员 PowerShell 中执行：

```powershell
.\Manage-LabAccounts.ps1
```

工具启动后会弹出可视化界面，包含七个功能标签页：

- **成员管理**：查看所有组成员及其所属导师组，支持一键启用/禁用账户、从组中移除、重置密码、修改分组。
- **创建账户**：输入用户名、密码并选择导师，一键创建账户，自动加入全员组和导师组、创建隔离的个人目录。
- **分组管理**：查看各导师组成员列表，新建导师组（自动创建文件夹和安全组），在组间调动成员。
- **性能优化**：解决非管理员账户桌面卡顿问题，一键应用 UAC 降级、组策略刷新优化、RemoteFX 开放等。
- **批量操作**：批量创建账户，每行格式为 `用户名,显示名称,导师名`。
- **离校管理**：学生毕业或离开课题组时，一键执行离校流程（禁用账户→归档数据→移除分组→生成交接清单）。
- **公告推送**：管理员发送通知，所有在线用户的悬浮窗会弹出提醒。

工具内置了操作日志，所有操作均有记录，方便追溯。

#### 6.2 命令行操作（备选）

如果需要在终端中快速操作，也可使用以下命令：

```powershell
# 新增成员（指定导师组）
net user 新用户名 初始密码 /add
net localgroup Lab_All 新用户名 /add
net localgroup Lab_张老师 新用户名 /add

# 禁用账户（毕业等）
net user 该用户名 /active:no

# 从某个导师组移除
net localgroup Lab_张老师 该用户名 /delete

# 从全员组移除（失去所有共享访问权限）
net localgroup Lab_All 该用户名 /delete

# 重新启用账户
net user 该用户名 /active:yes
```

#### 6.3 定期备份

建议配置 Windows Server 的"文件历史记录"或使用第三方工具定期备份 `D:\GroupData` 到外部存储。至少每周一次全量备份。

```powershell
# 使用 robocopy 做增量备份示例
robocopy D:\GroupData E:\Backup\GroupData /MIR /R:3 /W:5 /LOG:E:\Backup\backup.log
```

### 七、软件安装与系统维护规范

#### 7.1 软件安装分区

工作站上的软件按用途分为三档，安装位置不同：

**第一档：公共软件（管理员安装）**

所有人都要用的工具，如 PyCharm、Git、uv、数据库客户端、Office 等。统一安装到默认的 `C:\Program Files\` 或 `C:\Program Files (x86)\`，不要改安装路径。标准用户没有这些目录的写权限，因此只有管理员能安装，保证了软件的统一和稳定。

判断标准：如果你需要问"这个要不要给所有人用"，那大概率是公共软件，找管理员装。

**第二档：个人工具（用户自行安装）**

只有你自己用的小工具、特定版本的库、实验性的软件等。统一安装到 `D:\Users\你的用户名\Tools\` 下面。

```powershell
# 示例：安装一个便携版工具
# 下载解压到 D:\Users\zhangsan\Tools\notepad-plus\
# 然后在 Tools 文件夹里找到 exe 运行即可
```

注意事项：不要往 `C:\Users\你的用户名\AppData\` 里塞东西，那个目录会被各种软件的缓存自动占用，不适合手动管理。也不要往 `D:\GroupData\` 里装任何程序 — 共享数据盘只放数据，程序文件的小文件读写和锁文件操作在共享文件夹里容易出权限问题。

**第三档：Python 虚拟环境**

前面已详述。每个项目一个 `.venv`，通过 uv 管理，不污染全局。

#### 7.2 系统盘空间管理

多人工作站最大的隐患是 C 盘被撑爆。即使个人目录已重定向到 D 盘，Windows Update 缓存、各软件的临时文件仍会持续写入 C 盘。管理员应配置以下自动维护：

```powershell
# 每周自动清理临时文件（配置为计划任务，每周日 3:00 执行）
$action = New-ScheduledTaskAction -Execute "PowerShell.exe" `
    -Argument '-Command "Remove-Item $env:TEMP\* -Recurse -Force -ErrorAction SilentlyContinue; Clean-RecycleBin -Force -ErrorAction SilentlyContinue"'
$trigger = New-ScheduledTaskTrigger -Weekly -DaysOfWeek Sunday -At 3:00AM
Register-ScheduledTask -TaskName "WeeklyTempCleanup" -Action $action -Trigger $trigger -RunLevel Highest
```

建议每月检查一次 C 盘空间使用情况。如果低于 20% 可用空间，需要清理 Windows Update 缓存（`Dism /Online /Cleanup-Image /StartComponentCleanup`）或扩容。

**日志自动轮转**：系统已配置日志自动管理，审计日志超过 10MB 时自动按月归档，系统监控日志超过 5MB 时自动轮转。管理员无需手动管理日志文件。

#### 7.3 远程桌面会话管理

为避免用户断开连接后会话继续占用内存（导致其他人可用资源变少），建议配置会话超时策略：

```powershell
# 通过组策略设置：断开连接的会话 2 小时后自动注销
$tsPath = "HKLM:\SOFTWARE\Policies\Microsoft\Windows NT\Terminal Services"
if (-not (Test-Path $tsPath)) { New-Item -Path $tsPath -Force | Out-Null }
Set-ItemProperty -Path $tsPath -Name "MaxDisconnectionTime" -Value 7200000  # 2小时（毫秒）
Set-ItemProperty -Path $tsPath -Name "MaxIdleTime" -Value 14400000         # 4小时空闲超时
```

这条策略的含义是：断开连接超过 2 小时的会话自动注销，空闲超过 4 小时的会话也会被注销。

#### 7.4 系统托盘导航工具与悬浮球自助服务

工作站配备了一个系统托盘快捷导航工具（`Lab-TrayApp.ps1`），开机自动运行，常驻在桌面右下角。右键点击托盘图标即可快速打开：

- 我的个人文件夹（`D:\Users\你的用户名\`）
- 我的组内文件夹（`D:\GroupData\你的导师名\`）
- 全局公共文件夹（`D:\GroupData\_公共\`）
- **我的账户**（自助服务入口）

点击"我的账户"或通过悬浮球进入自助服务界面，你可以：

- **修改密码**：密码最少 8 个字符，无需联系管理员即可自行更改。
- **查看存储空间用量**：显示个人文件夹和组内文件夹的占用大小。
- **查看操作日志**：浏览与自己账户相关的近期操作记录。

这个工具也是一个小提醒：你正在使用一台公共工作站，请注意数据的存放位置。

#### 7.5 资源使用规范

多人同时使用时，请自觉遵守以下规范：

- **CPU 密集型任务**（大规模数据处理、模型训练等）：设置进程优先级为"低于正常"，或限制线程数。在 Python 中可通过环境变量控制：`$env:OMP_NUM_THREADS = 4`。
- **GPU 任务**（深度学习训练）：限制显存用量，避免独占 GPU。PyTorch 中可设置 `torch.cuda.set_per_process_memory_fraction(0.5)`。
- **长时间运行的任务**：建议使用 `tmux`（WSL 下）或 Windows 任务计划程序，避免因远程桌面断连导致任务中断。
- **禁止行为**：禁止运行挖矿程序、禁止修改他人进程、禁止关闭或修改系统监控服务。

**后台资源监控**：工作站运行有后台监控服务，持续跟踪 CPU、内存、GPU 和磁盘使用情况。如果某个用户的任务占用资源过高，系统会自动通知管理员。请注意合理安排长时间运行的任务（超过 48 小时的进程），避免长期独占大量资源影响他人使用。

#### 7.6 公告通知系统

管理员可以通过管理工具的"公告推送"标签页向所有在线用户发送广播通知，通知会以弹窗形式出现在所有用户的桌面上。

通知分为三个重要等级：

| 等级 | 说明 | 行为 |
|------|------|------|
| 普通 | 一般信息通知 | 显示为系统托盘气泡通知（右下角弹出，5秒后消失） |
| 重要 | 需要注意的事项 | 显示为系统托盘气泡通知（右下角弹出，5秒后消失），带警告图标 |
| 紧急 | 紧急事项（如系统维护） | 显示为桌面弹窗对话框（停留直到用户点击关闭） |

收到通知时请注意查看内容，尤其是标记为"紧急"的通知，通常涉及系统维护或重要变更。
