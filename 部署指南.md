## 课题组工作站 · 部署指南

本文档面向管理员，说明整套系统的架构、首次部署流程和日常运维方式。
系统已从 PowerShell 脚本重写为 .NET 8 WinForms / Worker Service 应用，部署与运维统一由 `LabWorkstation.Admin` 管理面板驱动。

---

### 一、系统架构

| 项目 | 类型 | 用途 |
|------|------|------|
| `LabWorkstation.Admin` | WinForms exe | 管理员面板（8 个 Tab），含系统初始化、账户/分组/公告/离校管理，首次运行自部署到 `C:\Scripts\LabWorkstation.Admin` |
| `LabWorkstation.TrayApp` | WinForms exe | 用户端悬浮导航 + 系统托盘，开机自启；含「我的账户」自助服务（改密/查用量/审计记录）与通知轮询 |
| `LabWorkstation.Monitor` | Worker Service exe | 后台资源监控守护进程，注册为开机自启计划任务（SYSTEM 身份） |
| `LabWorkstation.Kiosk` | WinForms exe | 自助开户终端，部署为 kiosk 账户的自定义 Shell，开机自动登录 |
| `LabWorkstation.Common` | 类库 | 共享内核：配置/审计/账户/ACL/通知/存储/部署器 |

**部署产物位置**（均由 Admin 面板自动管理）：

| 组件 | 部署路径 |
|------|----------|
| Admin 管理面板 | `C:\Scripts\LabWorkstation.Admin\LabWorkstation.Admin.exe` |
| TrayApp 悬浮导航 | `C:\Scripts\LabWorkstation.TrayApp\LabWorkstation.TrayApp.exe`（+ 公共启动目录快捷方式） |
| Monitor 监控服务 | `C:\Scripts\LabWorkstation.Monitor\LabWorkstation.Monitor.dll`（计划任务 `LabResourceMonitor`） |
| Kiosk 自助开户 | `C:\Scripts\LabWorkstation.Kiosk\LabWorkstation.Kiosk.exe`（kiosk 账户自定义 Shell） |
| 壁纸 | `C:\Scripts\LabWallpaper.png` |
| 配置/审计/监控日志 | `C:\ProgramData\LabWorkstation\`（`logs\`、`advisors.json`、`users.json`、`kiosk_queue\`） |

### 二、首次部署流程

#### 前置条件
- Windows Server 2019+ 或 Windows 10/11 Pro/Enterprise
- .NET 8 SDK 已安装（用于编译）
- 已以 Administrator 身份登录

#### 步骤 1：编译解决方案

在项目根目录执行：

```powershell
dotnet build LabWorkstation.slnx -c Debug
```

编译产物位于各项目的 `bin\Debug\net8.0-windows\` 目录。Admin 面板的部署器会自动从这些目录查找产物。

#### 步骤 2：启动 Admin 管理面板

在项目根目录以管理员身份运行：

```powershell
.\LabWorkstation.Admin\bin\Debug\net8.0-windows\LabWorkstation.Admin.exe
```

首次运行时，Admin 会**自部署**到 `C:\Scripts\LabWorkstation.Admin\` 并从该位置重启自身。后续会在 Administrator 桌面创建「工作站管理面板」快捷方式，双击即可启动（自动提权）。

> **测试模式**：加 `--test` 参数启动，所有操作仅作用于内存模拟状态，不修改真实系统。用于熟悉界面或验证流程。

#### 步骤 3：系统初始化

进入 Admin 面板后，切换到 **「系统初始化」** Tab。可逐项执行，或直接点击底部 **「一键初始化全部」**。一键初始化按依赖顺序执行以下 14 步：

| 步骤 | 说明 |
|------|------|
| 初始化目录骨架 | 创建 `D:\GroupData` / `_公共` / `_notifications` / `99_归档` / `D:\Users` 等基础目录 |
| 创建全员组 | 创建 `Lab_All` 组并加入 `Remote Desktop Users`（课题组成员可 RDP 登录） |
| 配置 Profile 目录到 D 盘 | 将系统 `ProfilesDirectory` 设为 `D:\Users`，新建用户 Profile 存放数据盘 |
| 收紧硬盘根目录权限 | `C:\` 清理非内置账户残留；`D:\` 将 Authenticated Users 降为只读 |
| 配置数据区权限 | 初始化根/公共/个人目录根 ACL（`Lab_All` 隔离） |
| 创建 SMB 共享 | 确保共享 `GroupData` 指向 `D:\GroupData` |
| 修复 RDP 授权 | 改为每用户授权模式 + 清除宽限期时间炸弹 |
| 部署壁纸 | 复制 `wallpaper.png` 到 `C:\Scripts\LabWallpaper.png`，设置当前会话/Default User 壁纸并锁定组策略 |
| 创建桌面快捷方式 | 创建 4 个公共桌面快捷方式（个人/组内/公共文件夹 + 使用手册）与使用须知文件 |
| 保护桌面只读 | 将公共桌面设为只读，防止标准用户删除桌面项目 |
| 应用系统策略 | RDP 超时 / Windows Update / 禁用安装 / 磁盘配额 |
| 部署资源监控 | 复制 Monitor 到 `C:\Scripts\LabWorkstation.Monitor\`，注册开机自启计划任务（SYSTEM 身份） |
| 部署悬浮导航 | 复制 TrayApp 到 `C:\Scripts\LabWorkstation.TrayApp\`，在公共启动目录创建快捷方式 |
| 部署 Kiosk 自助开户 | 复制 Kiosk 应用 + 创建 kiosk 账户 + 配置自定义 Shell + 开机自动登录 |

> 单步失败会中断后续步骤（存在依赖关系），可在日志区查看错误后逐项重试。

#### 步骤 4：创建导师组与用户账户

初始化完成后，切换到 **「分组管理」** Tab 创建导师组（`Lab_[导师名]`），再到 **「创建账户」** Tab 添加用户。创建账户会自动：建用户 → 加 `Lab_All` + 导师组 → 建个人目录 → 配 ACL。

#### 步骤 5（可选）：配置备份

备份功能尚未集成进 Admin 面板，仍使用独立脚本。以管理员身份运行：

```powershell
.\Setup-Backup.ps1
```

配置三级备份策略（基于 `robocopy`，无外部依赖）：

| 级别 | 内容 | 频率 |
|------|------|------|
| 一级 | 审计日志与系统配置 | 每日自动 |
| 二级 | 组内数据（GroupData） | 每周增量 |
| 三级 | 用户个人数据（Users） | 手动触发 |

- 备份目标默认 `E:\LabBackup\`（可在脚本顶部配置区修改）
- 手动备份：`C:\Scripts\Lab-BackupNow.ps1`
- 查看状态：`Get-BackupStatus`

#### 步骤 6（可选）：启用 BitLocker

```powershell
Enable-BitLocker -MountPoint 'D:' -EncryptionMethod Aes256 -RecoveryPasswordProtector
```

#### 步骤 7（建议）：重启服务器

重启后系统会自动以 kiosk 身份登录并显示自助开户界面。管理员可通过远程桌面以管理员身份登录（会覆盖 kiosk 自动登录）。

### 三、日常运维

所有日常操作通过 **LabWorkstation.Admin** 面板完成，8 个 Tab 覆盖全部场景：

| Tab | 功能 |
|-----|------|
| **系统初始化** | 分步或一键执行初始化；可单独重跑某一步（如重新部署 Monitor） |
| **成员管理** | 查看/筛选所有用户，启用/禁用/移除/重置密码/修改分组 |
| **创建账户** | 选导师 + 填信息 → 一键创建（建用户/加组/建目录/配权限） |
| **分组管理** | 新建/删除导师组，在组间调动成员 |
| **性能优化** | 一键应用非管理员账户的桌面优化策略 |
| **批量操作** | 格式 `用户名,中文名,导师名`，一次建好几个人 |
| **离校管理** | 禁用账户 → 归档个人数据到 `99_归档\` → 从所有安全组移出 → 生成离组交接清单 |
| **公告推送** | 编写通知 → 保存到 `pending\` → TrayApp 每 15 秒轮询弹窗推送；紧急通知被管理员删除后自动关闭 |

启动方式：双击 Administrator 桌面的「工作站管理面板」快捷方式（自动提权），或直接运行 `C:\Scripts\LabWorkstation.Admin\LabWorkstation.Admin.exe`。

### 四、资源监控

`LabWorkstation.Monitor` 作为计划任务 `LabResourceMonitor` 在系统启动时自动运行（SYSTEM 身份），每 **60 秒**采集一次指标，触发告警后 **10 分钟**内不重复报警：

- **CPU** > 90% → 告警
- **内存** > 90% → 告警
- **磁盘** > 90% → 告警
- **GPU** > 95%（仅在检测到 `nvidia-smi` 时启用）→ 告警
- **长时进程** — 运行超过 48 小时的进程单独列出
- 每 **10 分钟**输出一次状态摘要

日志路径：`C:\ProgramData\LabWorkstation\logs\system_monitor.log`（5 MB 自动轮转）。

管理命令：
```powershell
Get-ScheduledTask -TaskName LabResourceMonitor   # 查看任务状态
Start-ScheduledTask -TaskName LabResourceMonitor # 立即启动
```

### 五、用户自助服务

#### 普通用户（通过 TrayApp 悬浮导航）

用户登录后桌面右下角自动出现悬浮窗。点击悬浮窗或托盘菜单中的「**我的账户**」可进入自助页面：

- **修改密码** — 最低 8 位，支持复杂度校验
- **查看存储用量** — 显示个人目录占用空间
- **审计记录** — 查看自己的操作历史
- **接收公告** — 管理员推送的通知自动弹窗

#### 新用户自助开户（通过 Kiosk 终端）

未登录的访客可在物理机前使用 kiosk 终端自助开户：

1. 开机后自动以 kiosk 身份登录，进入 Kiosk 应用（无桌面、无其他系统功能）
2. 填写姓名、选择导师 → 提交开户请求
3. 请求写入 `C:\ProgramData\LabWorkstation\kiosk_queue\requests\`
4. 管理员在 Admin 面板审批后，账户信息写入 `responses\`
5. Kiosk 终端轮询取回结果，显示账户名和初始密码

> kiosk 账户使用随机强密码，自定义 Shell 仅运行 Kiosk 应用，DefaultPassword 注册表值 ACL 已收紧（仅 Administrators/SYSTEM 可读）。

### 七、安全组约定

| 组名 | 成员 | 权限 |
|------|------|------|
| `Lab_All` | 全体课题组成员 | 访问 `_公共` 区，RDP 登录 |
| `Lab_[导师名]` | 该导师组学生 | 访问对应导师组目录 |
| `kiosk` | 自助开户终端账户 | 仅运行 Kiosk 应用，无桌面访问 |

### 八、重新部署与升级

修改 C# 代码后，重新编译并部署单个组件：

1. `dotnet build LabWorkstation.slnx -c Debug`
2. 启动 Admin 面板
3. 在「系统初始化」Tab 单击对应组件的部署按钮（如「部署资源监控」重新部署 Monitor）

Admin 面板每次启动时会自部署到 `C:\Scripts\LabWorkstation.Admin`（仅在版本更新时覆盖）。
